# ResumeWebsite agent instructions

Vue 3 + Vite single-page resume site in `client/`, deployed to production at https://brandenimmerzeel.au by Vercel from `master`.
See `README.md` for the full architecture and the OpenSpec workflow.

## Branch discipline

- All work happens on `develop`: features, fixes, tests, docs, everything.
  If you find yourself on `master`, switch to `develop` before changing anything.
- Never commit, merge, or push to `master` directly.
  Changes reach `master` only through a release pull request from `develop` that a human reviews and merges.
  GitHub enforces this with the `protect-master` branch ruleset: direct pushes, force pushes, and branch deletion on `master` are rejected for everyone, admins included.
- Delivery shapes in this repo: /commit pushes finished work to `develop`; /release cuts a production release via the project release skill in `.claude/skills/release/`.
- If a hotfix ever lands on `master`, merge `master` back into `develop` immediately so the branches do not drift.

## Quality gate

- Typecheck and build: `npm run build` in `client/` (runs `vue-tsc` then Vite).
- Lint: `npm run lint` in `client/` (ESLint, zero warnings allowed).
- Tests: `npm run test:e2e` in `client/` (Playwright, starts its own dev server; first run may need `npx playwright install chromium`).
- Dependency audit: `npm audit` in `client/`.
- Run /patrol before committing to `develop` and always before opening a release PR.

## Skills

- `.claude/skills/openspec-*` are generated by the OpenSpec CLI; do not hand-edit them, regenerate via `openspec` instead.
- `.claude/skills/release/` is hand-written and owns the release process; it overrides the global /release skill for this repo.
